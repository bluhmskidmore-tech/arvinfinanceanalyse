## 背景上下文

### W3.3 已经建立的 adapter 骨架

`pnlAttributionAdapter.ts` 导出了 `derivePnlDataSectionState({ meta, isLoading, isError, errorMessage, isEmpty })` 函数，返回 `DataSectionState`。本任务使用它给每个 tab 派生 state。

### 核心 mechanical fix 模式

Contracts.ts 里 PnL 类型字段已从 `number` 升级为 `Numeric`。组件里的所有 `item.xxx`（xxx 是升级后的 Numeric 字段）access 都会 typecheck fail。Pattern：

| 原代码 | 改造后 | 备注 |
|---|---|---|
| `item.current_pnl` （预期 number 运算） | `(item.current_pnl.raw ?? 0)` | null-safe default 0 |
| `item.pnl_change != null` | `item.pnl_change != null` | 外部 null check 保留 |
| `(item.pnl_change / 1e8).toFixed(2)` | `((item.pnl_change.raw ?? 0) / 1e8).toFixed(2)` | 不动 `/1e8 + toFixed` |
| `item.volume_effect / 1e8` | `(item.volume_effect.raw ?? 0) / 1e8` | |
| `item.current_yield_pct != null ? \`${item.current_yield_pct.toFixed(2)}%\` : "—"` | `item.current_yield_pct != null ? \`${(item.current_yield_pct.raw ?? 0).toFixed(2)}%\` : "—"` | |
| `Math.abs(item.recon_error / 1e8)` | `Math.abs((item.recon_error.raw ?? 0) / 1e8)` | |
| `(item.current_scale ?? 0) / 1e8` | `(item.current_scale.raw ?? 0) / 1e8` | 注意 current_scale 现在非 nullable（Numeric 而非 Numeric \| null），但 `.raw` 可能 null |

**规则总结：** 凡是原来 `item.<numeric_field>` 视作 number 运算的，都改为 `(item.<numeric_field>.raw ?? 0)` 或 `(item.<numeric_field>?.raw ?? 0)`（前者 field non-null、后者 nullable）。其余 `.toFixed` / `/1e8` / ECharts series / Math.abs 等计算链保持不动。

### 组件新 interface 模式

每个组件从：

```tsx
type Props = { data: <Payload> | null };
```

改为：

```tsx
type Props = {
  data: <Payload> | null;
  state: DataSectionState;
  onRetry: () => void;
};
```

组件内部把原有 `if (!data) return <EmptyCard />` 替换为用 `DataSection` 包裹：

```tsx
return (
  <DataSection title="..." state={state} onRetry={onRetry}>
    {/* 原 body（已做 mechanical fix） */}
  </DataSection>
);
```

Import `DataSection` from `"../../../components/DataSection"`，`DataSectionState` from `"../../../components/DataSection.types"`。

### PnlAttributionView 核心改造

引入 adapter 构造每个 tab 的 state 派生：

```tsx
import { derivePnlDataSectionState } from "../adapters/pnlAttributionAdapter";

// 在 PnlAttributionView 内：
const volumeRateState = derivePnlDataSectionState({
  meta: volumeRateMeta,
  isLoading: loading && activeTab === "volume-rate",
  isError: error !== null && activeTab === "volume-rate",
  errorMessage: error,
  isEmpty: !volumeRateData || volumeRateData.items.length === 0,
});

// 类似 tplMarketState / compositionState / advancedSummaryState 等

// 传给组件：
<VolumeRateAnalysisChart data={volumeRateData} state={volumeRateState} onRetry={() => void loadData()} />
```

每个 tab 渲染时去掉原来 `if (loading) return ...` / `if (error) return ...` 判断（这些改由 DataSection 处理），直接传 state 到组件。

export function <ComponentName>({ data, state, onRetry }: Props) {
