### real 分支

```ts
getHomeSnapshot: async (options?: GetHomeSnapshotOptions) => {
  const params = new URLSearchParams();
  if (options?.reportDate) params.set("report_date", options.reportDate);
  if (options?.allowPartial) params.set("allow_partial", "true");
  const qs = params.toString();
  const response = await fetchImpl(
    `${baseUrl}/ui/home/snapshot${qs ? `?${qs}` : ""}`,
    { headers: { Accept: "application/json" } },
  );
  if (!response.ok) {
    throw new Error(`Request failed: /ui/home/snapshot (${response.status})`);
  }
  return (await response.json()) as ApiEnvelope<HomeSnapshotPayload>;
},
```

### Step 5：DashboardPage.tsx 切到 snapshot

核心改动：用单个 `snapshotQuery` 代替两个 `overviewQuery` / `pnlQuery`。

顶部 import 增加：
```tsx
import type { HomeSnapshotPayload } from "../../../api/contracts";
```

替换原来的 `overviewQuery` / `pnlQuery` 部分为：

```tsx
const [allowPartial, setAllowPartial] = useState(false);
const snapshotQuery = useQuery({
  queryKey: ["home-snapshot", client.mode, requestedDateLabel, allowPartial],
  queryFn: () =>
    client.getHomeSnapshot({
      reportDate: reportDate || undefined,
      allowPartial,
    }),
  retry: false,
});
```

用一个 useMemo 把 snapshot 拆成两个 envelope-like：

```tsx
const { overviewEnv, attributionEnv } = useMemo(() => {
  const env = snapshotQuery.data;
  if (!env) return { overviewEnv: undefined, attributionEnv: undefined };
  return {
    overviewEnv: {
      result_meta: env.result_meta,
      result: env.result.overview,
    },
    attributionEnv: {
      result_meta: env.result_meta,
      result: env.result.attribution,
    },
  };
}, [snapshotQuery.data]);

const adapterOutput = useMemo(
  () =>
    adaptDashboard({
      overviewEnv,
      attributionEnv,
      overviewLoading: snapshotQuery.isLoading,
      overviewError: snapshotQuery.isError,
      attributionLoading: snapshotQuery.isLoading,
      attributionError: snapshotQuery.isError,
    }),
  [overviewEnv, attributionEnv, snapshotQuery.isLoading, snapshotQuery.isError],
);
```

原有的 `overviewMeta` / `attributionMeta` / `attentionItems` / `overviewStatus` / `attributionStatus` / `overviewDateLabel` / `attributionDateLabel` 逻辑：保留，改为从 `adapterOutput.overview.meta` / `adapterOutput.attribution.meta` 取。

**删除**原来的 `resolveReturnedDateLabel` 中关于"effective_report_dates 多日聚合到 mixed" 的分支 —— 在 snapshot 模式下，四域都有同一 `report_date`，`mixed` 不应再出现。但保留 `mixed` 作 fallback（partial mode 可能导致 mixed）。

加一个 UI：在 header `actions` 里，除了 date picker 外，加一个 "允许历史日（含缺域）" 复选框，绑定 `allowPartial` state。

snapshot 特有的额外信息展示：当 `snapshotQuery.data?.result.mode === "partial"` 或 `domains_missing.length > 0` 时，在 attention strip 里追加一条 "该日部分业务域不可用: X, Y, Z"。

其他区域（FormalResultMetaPanel、OverviewSection 调用、PnlAttributionSection 调用）保持 W2.5/W2.6 的样子（消费 adapterOutput.overview / .attribution）。
