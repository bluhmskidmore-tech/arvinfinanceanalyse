## 背景上下文

### W3.1 现状

`backend/app/schemas/pnl_attribution.py` 的 17 个 pydantic class 每个都有：

```python
_NUMERIC_FIELDS: ClassVar[dict[str, tuple[NumericUnit, bool]]] = {
    "field_x": ("yuan", True),
    ...
}

@model_validator(mode="before")
@classmethod
def _coerce(cls, data: Any) -> Any:
    return _apply_numeric_coercion(cls._NUMERIC_FIELDS, data)
```

service 层目前依赖这个 validator 把 `pa_wb.build_*` 返回的 float-dict 自动 coerce 成 Numeric-dict。W3.2 的目标是**服务层显式构造 Numeric-dict**，之后 validator 只做幂等校验，不再承担"兜底"责任。

### `pa_wb.build_*` 的输出形态

`backend/app/core_finance/pnl_attribution/workbench.py` 返回的是 **float-dict**，例：

```python
{
  "current_period": "2025-04",
  "total_current_pnl": 3.21e9,           # 裸 float
  "total_previous_pnl": 3.00e9,
  ...
  "items": [
    {
      "category": "TPL",
      "current_scale": 1.0e11,           # 裸 float
      "current_pnl": 1.25e9,
      ...
    },
    ...
  ],
  "has_previous_data": True,
}
```

**保持 `pa_wb` 边界**（不改 `core_finance/`；它产出 float 就产 float）。promotion 在 service 层完成。

### 9 个 envelope 函数

位于 `pnl_attribution_service.py`：

1. `volume_rate_attribution_envelope` → `VolumeRateAttributionPayload`
2. `tpl_market_correlation_envelope` → `TPLMarketCorrelationPayload`
3. `pnl_composition_envelope` → `PnlCompositionPayload`
4. `attribution_analysis_summary_envelope` → `PnlAttributionAnalysisSummary`
5. `carry_roll_down_envelope` → `CarryRollDownPayload`
6. `spread_attribution_envelope` → `SpreadAttributionPayload`
7. `krd_attribution_envelope` → `KRDAttributionPayload`
8. `advanced_attribution_summary_envelope` → `AdvancedAttributionSummary`
9. `campisi_attribution_envelope` → `CampisiAttributionPayload`

每个函数当前形态：

```python
payload = pa_wb.build_xxx(...)  # 返回 float-dict
p = PayloadClass.model_validate(payload).model_dump(mode="json")  # 依赖 W3.1 coerce
return build_formal_result_envelope(result_meta=..., result_payload=_with_optional_warnings(p, warn=...))
```

目标形态：

```python
payload = pa_wb.build_xxx(...)  # 仍返回 float-dict
promoted = _promote_payload_numerics(payload, PayloadClass)  # 显式 Numeric-dict
p = PayloadClass.model_validate(promoted).model_dump(mode="json")
return build_formal_result_envelope(result_meta=..., result_payload=_with_optional_warnings(p, warn=...))
```

### Promote helper 设计

`_promote_payload_numerics(payload: dict, PayloadClass: type[BaseModel]) -> dict` 用 `PayloadClass._NUMERIC_FIELDS`（class-var）把 float→Numeric-dict 转换；对 list-of-nested-model 字段（如 `items: list[VolumeRateAttributionItem]`）递归下钻，取 list item 对应 class 的 `_NUMERIC_FIELDS`。

利用 pydantic `PayloadClass.model_fields[name].annotation` 读嵌套类型，或直接**硬编码一张 class→item_class 映射**（更显式、更好维护）：

```python
# Service-local registry; 9 payload classes + 4 item classes recorded.
_LIST_FIELD_ITEM_CLASS: dict[tuple[type, str], type] = {
    (VolumeRateAttributionPayload, "items"): VolumeRateAttributionItem,
    (TPLMarketCorrelationPayload, "data_points"): TPLMarketDataPoint,
    (PnlCompositionPayload, "items"): PnlCompositionItem,
    (PnlCompositionPayload, "trend_data"): PnlCompositionTrendItem,
    (CarryRollDownPayload, "items"): CarryRollDownItem,
    (SpreadAttributionPayload, "items"): SpreadAttributionItem,
    (KRDAttributionPayload, "buckets"): KRDAttributionBucket,
    (CampisiAttributionPayload, "items"): CampisiAttributionItem,
}
```

单元级 helper 示例：

```python
from backend.app.schemas.common_numeric import Numeric, NumericUnit, numeric_from_raw
from backend.app.schemas.pnl_attribution import (
    AdvancedAttributionSummary,
    CampisiAttributionItem,
    CampisiAttributionPayload,
    CarryRollDownItem,
    CarryRollDownPayload,
    KRDAttributionBucket,
    KRDAttributionPayload,
    PnlAttributionAnalysisSummary,
    PnlCompositionItem,
    PnlCompositionPayload,
    PnlCompositionTrendItem,
    SpreadAttributionItem,
    SpreadAttributionPayload,
    TPLMarketCorrelationPayload,
    TPLMarketDataPoint,
    VolumeRateAttributionItem,
    VolumeRateAttributionPayload,
)


def _numeric_dict(raw: float | None, unit: NumericUnit, sign_aware: bool) -> dict:
    return numeric_from_raw(raw=raw, unit=unit, sign_aware=sign_aware).model_dump(mode="json")


def _promote_flat(payload: dict, NumericClass: type) -> dict:
    """Promote a single-level dict's fields declared in NumericClass._NUMERIC_FIELDS."""
    field_map: dict[str, tuple[NumericUnit, bool]] = getattr(NumericClass, "_NUMERIC_FIELDS", {}) or {}
    out = dict(payload)
    for name, (unit, sign_aware) in field_map.items():
        if name not in out:
            continue
        v = out[name]
        if v is None or isinstance(v, Numeric):
            continue
        if isinstance(v, dict) and {"raw", "unit", "display", "precision", "sign_aware"} <= set(v.keys()):
            continue  # already promoted
        if isinstance(v, (int, float)):
            out[name] = _numeric_dict(float(v), unit, sign_aware)
    return out


def _promote_payload_numerics(payload: dict, PayloadClass: type) -> dict:
    """Promote governed numeric fields of payload + all nested list items."""
    out = _promote_flat(payload, PayloadClass)
    for (cls, field_name), item_cls in _LIST_FIELD_ITEM_CLASS.items():
        if cls is not PayloadClass:
            continue
        lst = out.get(field_name)
        if not isinstance(lst, list):
            continue
        out[field_name] = [_promote_flat(it, item_cls) if isinstance(it, dict) else it for it in lst]
    return out
```

**注意：**

- `value is None` 要跳过（Optional 字段允许 null）
- 已经是 Numeric 实例或 5-key dict 要跳过（幂等）
- 只处理 `_NUMERIC_FIELDS` 声明的字段；其他字段（`category`, `bond_count: int`, string 字段）不动
- 不要 deep-copy；浅 copy + list-comprehension 即可

### 为什么不完全删 W3.1 coerce validator？

保留 validator 的好处：防御性（万一有新 caller 忘了走 promote，schema 层仍然能接住）。**但** service 层的测试必须显式证明 "如果 coerce 被临时跳过，service 输出仍然合法"，这样 coerce validator 就从"必须"退化为"defense-in-depth"。

本 task **不删** W3.1 的 `_coerce_value_to_numeric`；验证策略见 Step 3 新测试 `test_pnl_attribution_service_explicit_numeric.py`。
