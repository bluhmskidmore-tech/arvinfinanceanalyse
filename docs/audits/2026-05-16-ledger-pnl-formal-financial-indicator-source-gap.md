## 结论

`ledger-pnl` 现在展示的是两条不同链路：

1. QDB 总账+日均分析链路：可复算总资产、总负债、贷款、存款、投资、存贷比、贷款减值准备率、定期化率、活期率、高流动性占比。
2. 正式财务指标链路：集团营业收入、业务及管理费、减值损失、利润总额、净利润、归母净利润、成本收入比、集团 ROA、集团 ROE、正式拨贷比仍缺正式来源。

本次审计判断：不要把 QDB 分析链路直接提升为正式财务指标。Excel 的正式指标包含集团合并范围、子公司、拨备余额、核销/收回、利润表管理口径和部分硬编码输入，当前系统没有完整可治理来源。

## Excel 指标到系统来源映射

| 指标 | 当前是否可正式落地 | 当前可用系统来源 | 缺口判断 |
| --- | --- | --- | --- |
| 集团营业收入 | 不可 | QDB 只能给母公司部分营收分项，且部分分项缺失 | 需要集团合并营收来源、子公司营收、抵销/管理调整；不能用 QDB 已有部分加总冒充 |
| 母公司营收（并表口径） | 不可完整落地 | 贷款利息收入、存款利息支出等部分可由 QDB 复算 | 缺金融投资利息收入、同业净收入、非息明细、FTP/营收分项来源 |
| 业务及管理费 | 不可 | 当前 QDB 月度分析没有正式费用来源模型 | Excel 计算表为硬编码/外部输入；需正式费用来源或管理报表输入 |
| 减值损失 | 不可 | QDB 可看到总账损益科目和贷款减值准备率分析值 | Excel 减值损失包含贷款减值损失、其他资产减值、子公司/村镇银行；需拨备/核销/收回来源闭环 |
| 利润总额 | 不可 | formal PnL 有投资持仓损益，但不是集团利润表 | 需要正式利润表口径；不能用 `fact_formal_pnl_fi.total_pnl` 替代集团利润总额 |
| 净利润 | 不可 | 无集团所得税/净利润正式来源 | 需要集团利润表或税费/调整来源 |
| 归母净利润 | 不可 | 无少数股东损益/归母调整来源 | 需要归母口径来源，至少包括子公司权益比例和少数股东损益 |
| 成本收入比 | 不可 | 公式 `业务及管理费 / 集团营业收入` 已明确 | 分子分母均未正式接入；公式可实现，但必须等待正式收入和费用值 |
| 集团 ROA | 不可 | 公式依赖净利润和集团总资产平均值 | 净利润和集团总资产正式口径未接入；QDB 总资产不是集团总资产 |
| 集团 ROE | 不可 | Excel 公式存在，但依赖多个未解释项 | 需要权益、归母、其他调整项来源；当前无法治理 |
| 集团总资产 | 不可 | QDB 总资产 202603 为 8144.05 亿元 | Excel 集团总资产为 8342.02547 亿元，差 197.98 亿元；需要合并范围和调整项 |
| 贷款余额（母公司） | 候选可落地 | QDB 202603 贷款总额为 4189.47 亿元 | 与 Excel 母公司贷款余额 4189.46747 亿元对齐；可作为第一批 golden sample 候选，但仍需口径批准 |
| 存款余额（母公司） | 不可直接正式落地 | QDB 202603 存款总额为 5115.96 亿元 | Excel 母公司存款余额为 5120.63810 亿元，差 4.68 亿元；需定位排除/补充科目或调整项 |
| 拨贷比 | 不可 | QDB 贷款减值准备率为 2.70% | Excel 拨贷比为 2.9263%，分子是贷款减值准备余额 122.7296 亿元，且集团贷款余额为 4193.9954 亿元；不能用 QDB 科目 131 / 贷款总额替代 |
| 不良贷款率/拨备覆盖率 | 不可 | QDB 当前没有正式 NPL 和拨备覆盖率来源 | 需要不良贷款额、贷款减值准备余额、核销/收回来源 |

## 建议接入顺序

1. 锁 golden sample，不先扩页面
   - 把上表 202603 值固化为后端测试样本或文档化 golden sample。
   - 每个指标保存：指标名、report_month、值、单位、Excel sheet/cell、公式、口径状态。

2. 新增正式财务指标读模型
   - 建议新增 `backend/app/core_finance/formal_financial_indicators.py`。
   - 输出粒度：`report_month + metric_key + consolidation_scope + unit`。
   - 字段至少包括：`value`、`unit`、`basis`、`formal_use_allowed`、`source_version`、`rule_version`、`cell_ref/golden_sample_ref`、`missing_reason`。

3. 第一批只落地低风险候选
   - 候选：贷款余额（母公司），因为 QDB 202603 与 Excel 值对齐。
   - 存款余额先不落地为正式值，必须解释 4.68 亿元差异。
   - 集团总资产、拨贷比、ROA、ROE 不进入第一批。

4. 第二批接母公司营收正式源
   - 先补齐金融投资利息收入、同业资产负债净收入、非息明细和 FTP/营收调整。
   - 目标是让母公司营收 `40.5057623568` 可以由正式来源重建。

5. 第三批接集团合并和资产质量源
   - 补子公司、村镇银行、抵销/管理调整、拨备余额、不良贷款额、核销/收回。
   - 目标是集团营业收入、净利润、归母净利润、拨贷比、ROA、ROE 可正式计算。

6. 页面最后改为正式指标优先
   - `/ledger-pnl` 第一屏优先消费正式财务指标读模型。
   - QDB 月度分析继续作为解释性穿透，不冒充正式指标。
