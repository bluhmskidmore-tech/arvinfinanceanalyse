# MOSS 金融 Agent 论文写作包

本目录用于把 MOSS 系统整理成一篇面向中文核心/普刊的金融科技论文。

论文主线：

> 可治理金融 Agent 分析系统设计与验证：基于固定收益业务的证据链、指标契约与正式计算隔离框架

本文不把 MOSS 写成泛泛的“AI+金融系统开发”，而是聚焦一个更可发表的问题：

> 在金融分析场景中，如何让 Agent 的回答绑定正式计算、指标口径、数据血缘和审计证据，从而降低幻觉、口径漂移和不可复核风险。

## 文件说明

| 文件 | 用途 |
| --- | --- |
| `material-pack.md` | 从现有项目材料中提炼论文可用证据、可写主张和禁止夸大的边界。 |
| `paper-blueprint.md` | 论文题目、摘要、研究问题、创新点、目录、图表清单。 |
| `5.5pro-prompts.md` | 给 5.5PRO 使用的主写手提示词、章节提示词和审稿人提示词。 |
| `experiment-plan.md` | 最小可发表实验设计、任务集、指标口径和记录模板。 |
| `draft-skeleton.md` | 可直接扩写的论文正文骨架，每章带证据来源说明。 |

## 推荐写作流程

1. 先阅读 `material-pack.md`，确认哪些系统能力已经有项目证据，哪些不能夸大。
2. 把 `material-pack.md` 和 `paper-blueprint.md` 交给 5.5PRO，先生成“论文设计”，不要直接写全文。
3. 按 `draft-skeleton.md` 的章节顺序逐章生成正文。
4. 每章写完后，使用 `5.5pro-prompts.md` 里的“审稿人模式”自审并重写。
5. 用 `experiment-plan.md` 补实验表格、案例表和结果记录。
6. 论文定稿前，再做一次“反夸大审查”：所有实验数据、系统能力和结论必须能追到项目材料或实验记录。

## 论文定位

推荐投稿定位：

- 中文金融科技、信息系统、管理信息化、金融工程应用类期刊。
- 文章类型优先选择“系统设计与应用验证”或“金融科技应用研究”。
- 不按纯 AI 算法论文写，除非后续新增模型、算法或可公开 benchmark。

## 写作硬约束

- 不允许让模型自由编造实验数据。
- 不允许把未验证的 Agent 能力写成正式金融结果。
- 不允许把 scenario 或 analytical 结果写成 formal 结果。
- 不允许把当前文档明确排除或延后的页面写成已完成闭环。
- 每个核心主张都必须有项目证据、实验记录或外部文献支撑。
