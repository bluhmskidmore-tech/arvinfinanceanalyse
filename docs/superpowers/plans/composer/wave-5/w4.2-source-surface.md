## 背景上下文

### 设计书 § 7.1 原文

> `ResultMeta` 新增必填字段 `source_surface: Literal["executive_analytical","formal_attribution","formal_pnl","formal_balance","formal_liability","bond_analytics","risk_tensor"]`
>
> 后端所有 envelope 构造强制带上该字段；CI lint 拒绝漏填。

**本 task 的工程折衷：**

- 直接 "必填" 会立刻让**所有**未迁移 service 坏掉（balance / liability / bond / risk 等还没做）。采用 **additive-only**：先设成 `Optional` + 默认 `None`
- 在 executive & pnl-attribution 两条链路上显式填值（这两条是本工程 M1/M2 已治理完的链路）
- 其他域的填值 + "拒绝漏填" 的 lint 放到 W5.7 统一做

### 当前 `ResultMeta` 形态（`backend/app/schemas/result_meta.py`）

```python
from datetime import datetime, timezone
from typing import Any
from typing import Literal

from pydantic import BaseModel, Field


class ResultMeta(BaseModel):
    trace_id: str = Field(...)
    basis: Literal["formal", "scenario", "analytical"] = "formal"
    result_kind: str = "analysis_view"
    formal_use_allowed: bool = True
    source_version: str
    vendor_version: str = "vv_none"
    rule_version: str
    cache_version: str
    quality_flag: Literal["ok", "warning", "error", "stale"] = "ok"
    vendor_status: Literal["ok", "vendor_stale", "vendor_unavailable"] = "ok"
    fallback_mode: Literal["none", "latest_snapshot"] = "none"
    scenario_flag: bool = False
    generated_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
    filters_applied: dict[str, Any] = Field(default_factory=dict)
    tables_used: list[str] = Field(default_factory=list)
    evidence_rows: int | None = None
    next_drill: list[str | dict[str, Any]] = Field(default_factory=list)
```

### `formal_result_runtime.py` 的 4 个 builder

- `build_analytical_result_meta`
- `build_scenario_result_meta`
- `build_formal_result_meta`
- `build_formal_result_meta_from_lineage`

它们底层都走 `_build_result_meta`。要改的是：

1. `_build_result_meta` 新增 kwarg `source_surface: SourceSurface | None = None`（其中 `SourceSurface = Literal[...]` 在该文件内定义或从 schema 导入）
2. 将其传到 `ResultMeta(...)` 实例化
3. 4 个公开 builder 同样加 kwarg 透传

### executive_service 与 pnl_attribution_service 的 envelope 构造点

- **executive_service.py**：grep `build_analytical_result_meta(` 和 `build_formal_result_meta(` 的调用点，每处加 `source_surface="executive_analytical"`
- **pnl_attribution_service.py**：9 个 envelope 函数里 `_meta_ok(...)` / `_meta_warn(...)` 用的是 `build_formal_result_meta(...)`；最干净的做法是给 `_meta_ok` / `_meta_warn` helper 函数加 `source_surface` 默认值 `"formal_attribution"`，然后 9 处 call site 不用改

## 实现步骤

### Step 1：改 `result_meta.py`

```python
from datetime import datetime, timezone
from typing import Any, Literal

from pydantic import BaseModel, Field


SourceSurface = Literal[
    "executive_analytical",
    "formal_attribution",
    "formal_pnl",
    "formal_balance",
    "formal_liability",
    "bond_analytics",
    "risk_tensor",
]


class ResultMeta(BaseModel):
    trace_id: str = Field(..., description="Trace identifier for governance and audit.")
    basis: Literal["formal", "scenario", "analytical"] = "formal"
    result_kind: str = "analysis_view"
    formal_use_allowed: bool = True
    source_version: str
    vendor_version: str = "vv_none"
    rule_version: str
    cache_version: str
    quality_flag: Literal["ok", "warning", "error", "stale"] = "ok"
    vendor_status: Literal["ok", "vendor_stale", "vendor_unavailable"] = "ok"
    fallback_mode: Literal["none", "latest_snapshot"] = "none"
    scenario_flag: bool = False
    generated_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
    filters_applied: dict[str, Any] = Field(default_factory=dict)
    tables_used: list[str] = Field(default_factory=list)
    evidence_rows: int | None = None
    next_drill: list[str | dict[str, Any]] = Field(default_factory=list)
    source_surface: SourceSurface | None = None
```

关键：

- `SourceSurface` 作为 module-level `TypeAlias`（或裸 `Literal` 赋值），以便其他文件 `from backend.app.schemas.result_meta import SourceSurface` 导入
- 默认值 `None`（additive）
- 位置放在字段末尾，避免搅乱既有构造参数顺序

### Step 2：改 `formal_result_runtime.py`

1. 顶部 import 追加：
   ```python
   from backend.app.schemas.result_meta import ResultMeta, SourceSurface
   ```

2. `_build_result_meta` 签名加 `source_surface: SourceSurface | None = None`，并在 `ResultMeta(...)` 实例化的末尾传 `source_surface=source_surface`。

3. 4 个公开 builder (`build_analytical_result_meta` / `build_scenario_result_meta` / `build_formal_result_meta` / `build_formal_result_meta_from_lineage`) 各加 `source_surface: SourceSurface | None = None` kwarg 并透传到 `_build_result_meta(...)`。注意 `build_formal_result_meta_from_lineage` 的实现里有一次对 `build_formal_result_meta(...)` 的调用，记得把 `source_surface` 透传过去。

4. `build_formal_result_envelope` / `build_formal_result_envelope_from_lineage` **不**要改签名（envelope builder 只是 `{result_meta, result}` 包装器，上游已经把 `source_surface` 填在 `result_meta` 里了）。

### Step 3：改 `executive_service.py`

grep `build_analytical_result_meta(` 与 `build_formal_result_meta(`（含其他拼写，例如 lineage 版本）的调用点：


每个命中点：kwarg 追加 `source_surface="executive_analytical"`。如果命中点很多（预计 10-20 处），**可以**考虑在文件顶部或"工具函数区"写一个 `_meta_executive(...)` 局部 helper，把 `source_surface="executive_analytical"` 锁死，然后让所有 call site 改走这个 helper。但这属于 refactor，会让 diff 胀——**更稳的做法是只在每个 call site 显式加 kwarg**；subagent 自行判断。

**注意：**`/ui/home/snapshot` 端点也走 executive_service（W4.1 已上线），该端点 envelope 同样应是 `executive_analytical`。

### Step 4：改 `pnl_attribution_service.py`

**最干净的改法：在这两个 helper 里 hard-code** `source_surface="formal_attribution"`：

```python
        ...
```
