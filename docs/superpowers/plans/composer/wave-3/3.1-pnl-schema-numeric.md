### 当前 `schemas/pnl_attribution.py` 结构

221 行，17 个 pydantic 类：

- `VolumeRateAttributionItem` / `VolumeRateAttributionPayload`
- `TPLMarketDataPoint` / `TPLMarketCorrelationPayload`
- `PnlCompositionItem` / `PnlCompositionTrendItem` / `PnlCompositionPayload`

- `CarryRollDownItem` / `CarryRollDownPayload`
- `SpreadAttributionItem` / `SpreadAttributionPayload`
- `KRDAttributionBucket` / `KRDAttributionPayload`
- `AdvancedAttributionSummary`
- `CampisiAttributionItem` / `CampisiAttributionPayload`

现在所有数字字段（`current_pnl: float` / `volume_effect: float | None` / `market_value: float` / `pnl_change_pct: float | None` 等）都是 `float` 或 `float | None`。非数字字段（`category: str` / `level: int` / `bond_count: int` / 数组 / bool 等）**保持不动**。

### 升级策略（与 W2.1 平行）

采用 additive union + coerce validator。核心 pattern：

```python
# 每个 class 顶部加：
from backend.app.schemas.common_numeric import Numeric, numeric_from_raw

# class 内字段类型 float → Numeric
# 字段类型 float | None → Numeric

# class 加 validator：
@model_validator(mode="before")
@classmethod
def _coerce_numbers(cls, data: Any) -> Any:
    if not isinstance(data, dict):
        return data
    out = dict(data)
    for field_name, unit_config in cls._numeric_fields().items():
        if field_name in out:
            out[field_name] = _coerce_numeric_field(out[field_name], unit_config)
    return out
```

**不过直接在每个 class 重复 17 份 validator 太冗长**。更干净：提供一个 module-level helper 函数 `_coerce_value_to_numeric(value, unit, sign_aware, precision)`，每个 class 的 validator 调用它。

### Coerce 规则

- `None` → `None`（schema 里如果字段是 `Numeric | None`，允许 null）
- `float` / `int` → `Numeric(raw=float(value), unit=<class 指定>, display=<from numeric_from_raw>, ...)`
- `dict` 含 `raw`/`unit`/`display`/`precision`/`sign_aware` 5 键 → 直接当 Numeric dict 让 pydantic validate
- `Numeric` 实例 → 直接使用
- 其他 → pydantic 自己报错

### 字段 unit / sign_aware 映射（关键决策，写在 code comment 里）

按字段名大类识别单位：

| 字段名 / 字段类 | unit | sign_aware | 备注 |
|---|---|---|---|
| `*_pnl` / `*_effect` / `*_return` / `*_income` / `*_gain` / `*_change` / `carry` / `carry_pnl` / `roll` / `rolldown` / `rolldown_pnl` / `static_pnl` / `static_return` | `yuan` | `True` | 金额（带符号） |
| `market_value` / `current_scale` / `previous_scale` / `total_market_value` / `market_value_start` | `yuan` | `False` | 规模（非负） |
| `ftp_rate` / `weighted_yield` / `coupon_rate` / `ytm` / `funding_cost` / `yield_change` / `treasury_change` / `spread_change` / `treasury_10y_*` / `avg_treasury_*` / `total_return_pct` / `*_pct` / `contribution_pct` | `pct` | `True` | 百分比/比率（都可能有符号） |
| `*_change_pct` / `correlation_coefficient` / `primary_driver_pct` | `pct` | `True` | |
| `duration` / `bucket_duration` / `portfolio_duration` | `ratio` | `False` | 年限（数字 + "年" 由 display 显示） |
| `tenor_years` | `ratio` | `False` | 年 |
| `weight` / `krd` | `ratio` | `False` | 小数 |
| `dv01` / `portfolio_dv01` / `treasury_10y_total_change_bp` | `dv01` / `bp` | `False` / `True` | 分别 |
| `curve_slope` | `ratio` | `True` | |
| `bond_count` / `total_actions` / `num_days` / `num_periods` | 保持 `int` | — | 不是 Numeric |

**完整映射在 code 里以常量或 helper 函数表达，不要逐字段判断。**

### 简化方案：按类 scoped 的 numeric_fields mapping

给每个 class 定义 class-level 常量 `_NUMERIC_FIELDS: dict[str, tuple[NumericUnit, bool]]`，validator 自动遍历：


Module-level helper：

```python
def _coerce_value_to_numeric(value: Any, unit: NumericUnit, sign_aware: bool) -> Any:
    """Coerce float / int / dict into Numeric-compatible dict.

    - None → None (caller must allow Optional)
    - Numeric instance / dict with 5 keys → passthrough
    - float / int → numeric_from_raw(raw=value, unit=unit, sign_aware=sign_aware)
    """
    if value is None:
        return None
    if isinstance(value, Numeric):
        return value
    if isinstance(value, dict) and {"raw", "unit", "display", "precision", "sign_aware"} <= set(value.keys()):
        return value
    if isinstance(value, (int, float)):
        return numeric_from_raw(raw=float(value), unit=unit, sign_aware=sign_aware).model_dump(mode="json")
    return value


def _apply_numeric_coercion(
    field_map: dict[str, tuple[NumericUnit, bool]],
    data: Any,
) -> Any:
    if not isinstance(data, dict):
        return data
    out = dict(data)
    for field_name, (unit, sign_aware) in field_map.items():
        if field_name in out:
            out[field_name] = _coerce_value_to_numeric(out[field_name], unit, sign_aware)
    return out
```

```python
"""Pydantic mirrors of frontend `contracts.ts` PnL attribution payloads.

W3.1: numeric fields migrated from bare ``float`` to ``Numeric``. A
module-level ``model_validator(mode="before")`` on each class automatically
coerces legacy ``float`` / ``int`` callsites into display-only ``Numeric``
dicts via ``numeric_from_raw``. Service layer (W3.2) will construct Numeric
directly; this transitional shim keeps pre-W3.2 callsites working.
"""
from __future__ import annotations

from typing import Any, ClassVar, Literal

from pydantic import BaseModel, model_validator

from backend.app.schemas.common_numeric import Numeric, NumericUnit, numeric_from_raw


def _coerce_value_to_numeric(value: Any, unit: NumericUnit, sign_aware: bool) -> Any:
    """Coerce float / int / dict into a Numeric-compatible structure.

    - ``None`` → ``None``
    - ``Numeric`` instance → pass through
    - ``dict`` with Numeric 5-key shape → pass through
    - ``float`` / ``int`` → coerced via ``numeric_from_raw`` and dumped to dict
    - other types → pass through (pydantic will raise)
    """
    if value is None:
        return None
    if isinstance(value, Numeric):
        return value
    if isinstance(value, dict) and {"raw", "unit", "display", "precision", "sign_aware"} <= set(value.keys()):
        return value
    if isinstance(value, (int, float)):
        return numeric_from_raw(
            raw=float(value), unit=unit, sign_aware=sign_aware
        ).model_dump(mode="json")
    return value


def _apply_numeric_coercion(
    field_map: dict[str, tuple[NumericUnit, bool]],
    data: Any,
) -> Any:
    if not isinstance(data, dict):
        return data
    out = dict(data)
    for field_name, (unit, sign_aware) in field_map.items():
        if field_name in out:
            out[field_name] = _coerce_value_to_numeric(out[field_name], unit, sign_aware)
    return out
```
