This file is navigation only and is non-authorizing. It always defers to
`AGENTS.md` -> `docs/DOCUMENT_AUTHORITY.md` -> `docs/CURRENT_EFFECTIVE_ENTRYPOINT.md`.
If this file conflicts with that chain, follow the authority chain.

## Purpose

This directory stores repo-visible planning history, scoped design notes,
implementation plans, audits, prompt packs, and UI specs. It does not grant
execution authority by itself.

## Use

- Read these files for context, rationale, and prior decisions.
- Confirm current authority through the repo-level read path before acting.
- Treat dated plans as supporting evidence unless a higher-priority instruction
  explicitly makes one active for the current lane.
