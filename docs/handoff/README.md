# Handoff Navigation

Status label: supporting

This file is navigation only and is non-authorizing. It always defers to
`AGENTS.md` -> `docs/DOCUMENT_AUTHORITY.md` -> `docs/CURRENT_EFFECTIVE_ENTRYPOINT.md`.
If this file conflicts with that chain, follow the authority chain.

## Purpose

This directory stores handoff notes for bounded workstreams. These files help
recover context across sessions, but they do not override the repo authority
chain or grant current execution scope.

## Use

- Use handoff files to understand prior work, blockers, and next-step context.
- Reconfirm any active scope through the repo-level read path before editing.
- Keep new handoffs scoped to one page, workflow, or bounded delivery lane.
